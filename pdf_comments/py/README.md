# 編集作業補助用ツール

## extract_pdf_comments_v2.py

PDFに埋め込まれた注釈（ハイライト・下線・波線・取り消し線・付箋コメント）を、
該当ページ番号（本のノンブル）付きでMarkdownまたはCSVに抽出するスクリプトです。

Googleドライブのプレビュー画面で付けるコメントではなく、**PDFファイル自体に
埋め込まれた注釈**（Adobe Acrobat、PDF-XChange、macOS プレビューなどで付けた
ハイライト・付箋）が対象です。

### できること

- ハイライト等の「対象文」と、それに紐づく「コメント本文」をセットで抽出
- 対象文はハイライトの頂点座標（Quad）ベースで抽出するため、複数行にまたがる
  ハイライトでも語順の乱れが少ない
- 見開き2ページが1PDFページに収まったレイアウト（左右にノンブルがある版）でも、
  単ページ1PDFページ=1印刷ページのレイアウトでも、実際の印刷ページ番号を自動判定
- 1ファイルずつの処理と、フォルダ内の複数PDFの一括処理の両方に対応
- 出力形式はMarkdown（Wordや原稿と並べて読みやすい）とCSV（Excelで管理したい場合）を選択可能

### セットアップ（初回のみ）

Python 3が必要です。以下のコマンドで依存パッケージをインストールしてください。

```bash
pip install pymupdf --break-system-packages
```

※ `--break-system-packages` が使えない・不要な環境（Windowsの通常インストールなど）では、
そのまま `pip install pymupdf` でも構いません。

### 使い方

#### 1ファイルだけ処理する

```bash
python extract_pdf_comments_v2.py 対象.pdf
```

同じフォルダに `対象.md` が生成されます。

出力先やファイル名を指定したい場合:

```bash
python extract_pdf_comments_v2.py 対象.pdf -o 出力先.md
```

CSVで出力したい場合:

```bash
python extract_pdf_comments_v2.py 対象.pdf --format csv
```

#### フォルダ内の複数PDFをまとめて処理する

フォルダのパスをそのまま渡すと、自動的にフォルダ内の全PDFを一括処理します
（PDFごとに、同名の `.md`（または `.csv`）が同じフォルダ内に生成されます）。

```bash
python extract_pdf_comments_v2.py "フォルダのパス"
```

明示的にバッチモードを指定したい場合は `--batch` も使えます（挙動は同じです）。

```bash
python extract_pdf_comments_v2.py --batch "フォルダのパス"
```

#### オプション一覧

| オプション | 説明 | 既定値 |
|---|---|---|
| `pdf_path` | 入力PDFファイル、またはPDFが入ったフォルダのパス | （必須） |
| `-o`, `--output` | 出力ファイルパス（単一ファイルモードのみ） | 入力ファイル名 + `.md`/`.csv` |
| `--format` | `md` または `csv` | `md` |
| `--footer-ratio` | ページ下部の何割からをノンブル検出対象にするか（0〜1） | `0.9`（下部10%） |
| `--batch` | 指定フォルダ内のPDFを一括処理 | — |

### 出力例（Markdown）

```markdown
# 対象

合計 3 件

## p.86

- **該当箇所**: 「エージェント」
  **コメント**: AIエージェント

## p.87

- **該当箇所**: 「る。」
  **コメント**: 他の項目と合わせて句点を取る
```

「該当箇所」が空欄の場合（付箋コメントなど、本文への紐づけがない注釈）は、
「（場所指定なし）」と表示されます。

### うまく抽出できないときは

- **ページ番号が「(PDFページN)」と表示される**
  ページ下部にノンブル（数字だけの単語）が見つからなかった場合のフォールバックです。
  表紙・扉ページなどノンブルがないページでは正常な挙動ですが、本文ページで頻発する
  場合は `--footer-ratio` の値を調整してみてください（例: `--footer-ratio 0.85`）。

- **該当箇所が空、または期待した文言と違う**
  ハイライトではなく付箋（テキストのみの注釈）にはそもそも「対象文」が存在しません。
  また、表組みや複雑なレイアウトをまたぐハイライトでは、抽出順序が本来の見た目と
  ずれることがあります。その場合はPDF側で該当ページを目視確認してください。

- **`'...' is no file` のようなエラーが出る**
  フォルダのパスをそのまま渡した場合に自動でバッチモードへ切り替わりますが、
  古いバージョンのスクリプトではこの自動判定がないため、単一ファイルとして
  開こうとしてエラーになります。最新版のスクリプトを使ってください。
