# pdf-comment-extractor.html

`extract_pdf_comments_v2.py`（Python版）の読み取り機能を、ブラウザ完結の単体HTMLファイルに
移植したものです。Pythonのインストールやパッケージ導入は不要で、このHTMLファイルを
ブラウザで開くだけで動作します。

## 特徴

- **通信が発生しない**：pdf.js本体をファイル内に直接埋め込んでいるため、CDNや外部サーバーへの
  アクセスは一切ありません。オフラインでも動作します
- ドラッグ&ドロップ、または選択で複数PDFをまとめて処理できます
- Python版と同じロジックで、ハイライト・下線・波線・取り消し線・付箋コメントを
  ページ番号（本のノンブル）付きで抽出します
- 結果はページ内にその場で一覧表示され、Markdown/CSVとして書き出せます

## 使い方

1. `pdf-comment-extractor.html` をダブルクリックしてブラウザで開く
2. PDFをドロップ、またはクリックしてファイルを選択（複数可）
3. 画面上に抽出結果が表示される
4. 「出力形式」でMarkdown/CSVを選び、各ファイルの「書き出し」ボタンでダウンロード

「ノンブル検出範囲」は、ページ下端から何%をノンブル(ページ番号)の探索範囲にするかの設定です。
Python版の `--footer-ratio` と同じ役割で、既定値は10%（下部1割）です。

## Python版との違い

| | Python版(extract_pdf_comments_v2.py) | HTML版(このツール) |
|---|---|---|
| 実行環境 | Python + PyMuPDF | ブラウザのみ |
| 書き込み・注釈追加 | 対応なし(読み取り専用) | 対応なし(読み取り専用) |
| バッチ処理 | `--batch フォルダ` | 複数ファイルを一度に選択/ドロップ |
| 該当箇所の抽出精度 | 頂点座標(Quad)ベース、実績あり | 同様のロジックで移植。**PDFの構造によっては数件単位の差異が出ることがある**（テキスト照合の実装が異なるため） |

抽出件数がPython版と完全一致しない場合がありますが、これは実装の違いによる想定内の誤差です。
どちらの結果も「だいたいこのあたり」を素早く把握するための下書きとして使い、最終的な該当箇所は
必要に応じてPDF側で目視確認してください。

## 既知の制限

- 処理はブラウザのメインスレッドで行われるため、非常にページ数の多いPDFでは処理中に
  タブが一瞬固まったように見えることがあります(処理自体は完了します)
- フォントによっては `translateFont failed` という警告がブラウザの開発者コンソールに
  出ることがありますが、抽出結果には影響しません

## ファイルサイズについて

pdf.js本体を内蔵しているため、ファイルサイズは約2.4MBあります。通常のテキストファイルと
比べると大きいですが、単体で完結して動作させるためのトレードオフです。
